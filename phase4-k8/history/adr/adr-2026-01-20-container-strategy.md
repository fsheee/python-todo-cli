# ADR-2026-01-20: Container Image Strategy for Phase 4\n\n## Status\nAccepted\n\n## Context\nBuild/deploy Phase 3 backend/frontend images to Minikube.\n\n## Decision\n- Backend: python:3.12-slim + uvicorn (multi-stage optional)\n- Frontend: node:20-alpine builder + nginx:alpine serve\n- Build local, `minikube image load` (no registry for local dev)\n- Tags: latest (dev), versioned for prod\n\n## Positive Consequences\n- Fast local dev (no push/pull)\n- Consistent with Phase 3 Dockerfiles\n- Minimal images (slim/alpine)\n\n## Negative Consequences\n- Not portable without registry\n- No multi-arch (arm64)\n\n## Alternatives\n- Docker Hub push/pull (slower local)\n- Minikube registry addon\n\nDate: 2026-01-20\nCo-Authored: Claude Sonnet 4.5\n\nStatus: Accepted"
