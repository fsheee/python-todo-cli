# ADR-2026-01-20: Helm Chart Design for Phase 4 K8s Deploy\n\n## Status\nAccepted\n\n## Context\nDeploy Phase 3 chatbot (frontend/backend/postgres) to local Minikube using Helm for reproducibility.\n\n## Decision\nSingle Helm chart `gordon/` with templates for:\n- frontend/backend deployments/services\n- postgres StatefulSet/service/PVC\n- ingress (NGINX host todo.local)\n- configmaps/secrets for env vars\n\nValues.yaml for customization (images, resources, postgres).\n\n## Positive Consequences\n- One-command deploy (`helm install`)\n- Namespace isolation (`todo-app`)\n- Resource limits/requests\n- Health checks/probes\n\n## Negative Consequences\n- Monolithic chart (future split if multi-env)\n- Minikube-local (not prod-ready secrets)\n\n## Alternatives\n- Separate charts per component (overkill for local)\n- Kustomize (less mature for beginners)\n\nDate: 2026-01-20\nCo-Authored: Claude Sonnet 4.5\n\nStatus: Accepted"
